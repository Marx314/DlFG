# Delta Tracker

Identifies new/gone developers and enriches the developer inventory with manager information from Azure AD.

## Overview

The delta tracker is a separate module that:
1. **Loads** the developer inventory CSV generated by `main.py`
2. **Queries** Azure AD for employee roster and manager hierarchy
3. **Identifies**:
   - New developers in inventory but not in Azure AD
   - Gone developers in Azure AD but inactive (no commits in 90+ days)
4. **Enriches** the CSV with:
   - Direct manager name
   - Full chain of command (manager > grandmanager > etc.)
   - Job title
   - Department

## Setup

### Prerequisites
- Azure AD tenant with Microsoft Graph API access
- Service principal credentials with:
  - `User.Read.All` permission
  - `Directory.Read.All` permission

### Configuration

Set environment variables:

```bash
export AZURE_TENANT_ID="your-tenant-id"
export AZURE_CLIENT_ID="your-client-id"
export AZURE_CLIENT_SECRET="your-client-secret"
export DAYS_INACTIVE=90  # Days to consider developer "gone"
```

Or add to `.env` file in parent directory.

## Running

```bash
# From project root
python -m delta_tracker.main

# Or from delta_tracker directory
cd delta_tracker
python main.py
```

The script automatically finds the latest developer inventory CSV.

## Output Files

Generated in `output/` and `output/delta_tracking/`:

### 1. Enriched CSV
- **Filename**: `YYYY-MM-developers-enriched.csv`
- **Location**: `output/`
- **Adds columns**:
  - `manager` - Direct manager name
  - `manager_chain` - Full hierarchy (e.g., "John Doe > Jane Smith > CEO")
  - `job_title` - Employee job title
  - `department` - Department/org

### 2. Delta Tracking
- **Filename**: `YYYY-MM-DD-delta.csv`
- **Location**: `output/delta_tracking/`
- **Rows**:
  - Status: NEW (developer in inventory but not in Azure)
  - Status: GONE (developer in Azure but inactive)
  - Date: When identified
  - Notes: Reason

### 3. Summary
- **Filename**: `YYYY-MM-DD-delta-summary.csv`
- **Statistics**:
  - Total developers in inventory
  - New developers count
  - Gone developers count
  - Total Azure AD users
  - Timestamp

## Integration

The enriched CSV can be used for:
- **Training programs**: Filter by manager or department
- **Reporting**: Cross-reference with training tool (e.g., Secure Code Warrior)
- **Compliance**: Track developer activity across platforms
- **Analytics**: Department-level security posture

## Troubleshooting

### Azure Authentication Fails
- Verify credentials are correct
- Check service principal has required permissions
- Ensure credentials are not expired

### No Developers Found in Inventory
- Run `main.py` first to generate inventory
- Check that inventory CSV exists in `output/` directory

### Manager Information Missing
- Developer email may not match Azure AD email
- User may not have manager assigned in Azure AD
- Check logs for specific user lookup failures
